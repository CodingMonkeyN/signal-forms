Key Takeaways

  • errors() - only this field's errors
  • errorSummary() - this field + ALL descendants
  • submit() handles the full submission lifecycle
  • Return errors from submit() to target specific fields
  • reset() clears touched/dirty without changing values
  • Combine touched() + invalid() for UX-friendly error display
